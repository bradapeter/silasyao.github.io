3.6 配送管理

“一块地”种植服务平台向顾客提供生鲜配送服务。配送有计划内的和计划外的。 计划内的配送是针对参与了合约计划（种植合约、宅配合约）的会员，每周将定期有果蔬物品配送到家，该配送计划相对，便于安排生产。 计划外的，则是会员在平台上购买的其它果蔬商品，该配送可以计划内的配送一起配送，达到降低物流成本，考虑到产品的特性（比如禽类、蛋类）可能需经其它专业物流进行配送，则配送不必和计划内的走同一物流，故无法通过一同配送降低成本。

3.6.1 配送服务说明

本节主要以现有黄陂木兰基地的现有业务（含将来可预见的）做如下约定，未来平台需满足多基地和多商家的配送和生产服务，在需要分析和设计时要考虑扩展需要，设计上要有一定的前瞻性。

种植服务合约中含有2个来源的蔬菜涉及配送：

  1. 专属菜地代耕代种定制蔬菜产出：每位会员私人专属30平米菜地，种植品种由顾客指定，在蔬菜进入采收期将根据成熟度分批分期进行采收和配送；
  2. 日常配套家常大众蔬菜食材包：应季蔬菜，每个包3.6斤，共100个包，每次一个包（蔬菜品种不限5种，品种随机），每周2次，持续保障家庭餐桌每日所需。

以上2种均为计划内配送内容，其中菜地面积、食材包的重量和数量只是举例进行说明，具体值会根据不同的种植服务产品包待定。

宅配合约，即没有专属菜地仅向顾客提供蔬菜宅配，如月度、季度的蔬菜宅配，相当于“种植合约”里的日常大众食材包，供应家庭餐桌每日所需。

此种合约，是否对所有用户开放，还是仅作为种植合约会员的增量补充，解决人口规模较大的家庭不够吃的问题，这个视企业发展和市场需求决定。此处的功能规划并不冲突。

针对参与了“种植服务合约”的家庭会员，“一块地”武汉黄陂木兰基地的配送服务说明如下：

  1. 每周定期将合约内的日常蔬菜配送到家，目前提供一周2次的同城宅配，配送集中在周二和周五配送，专属菜地定制产出的作物只有在成熟的时候才会有配送，其它时候只有日常配套家常蔬菜食材包送货上门；
  2. 配送范围集中在武汉中心城区及部分三环线外周边大型楼盘；
  3. 配送时段，在早上11点前送到（如无特殊要求，）。如果顾客需要晚间送货上门服务，需在配送计划中选择“19:00-22:00”时段；
  4. 宅配的货物在配送日前17:00可修改订单的配送地址，增加配送的品种和数量，也可停止此次配送。超过此时间请联系客服；
  5. 配送服务由合作物流公司或自营配送执行；
  6. 通过在幼儿园参与团购的顾客，在开园期间，幼儿园视同“自提点”，采用可循环回收装置进行配送，顾客可以接孩子放学时提货；寒暑假及小长假期间，则宅配到家；
  7. 计划外的配送（即会员在平台集市上另行购买和兑换的商品订单），如果是现货，则与最近的一次宅配合并进行配送。如果顾客有特殊要求，联系客服，可加急单独走物流，但应满足一定的条件，比如订单金额达到一定金额，或客户级别达到一定程度，否则需另收运费，在一定时间点之前提交的订单可以安排次日送或隔日送；
  8. 计划外的配送，如果是预售商品，待确定备货完成时间，再行与其相领的最近一次宅配合并进行；
  9. 订单执行情况及配送进展，系统应及时以手机短信和站内消息形式告知顾客，并在系统交易相关页面予以更新。

针对参与”宅配套餐合约“的家庭会员，其服务说明如下：

宅配套餐合约与种植合约内容主要区在于没有专属菜地，故无专属菜地定制蔬菜的产出和配送。仅有日常配套家常大众蔬菜食材包的配送，”宅配套餐合约“有几种规格，除每次配送的数量和品种有差异外，其它方面完全一样，其配送服务与“种植服务合约”的家庭会员**的完全一致。

针对普通个人散户（即通过卡券兑换生鲜商品），配送服务说明如下：

  1. 顾客现货订单，视平台和商家配送策略，进行配送，根据业务发展变化，早期配送同“合约”顾客一样，配送集中在周二或周五，配送范围集中在武汉中心城区及三环线外周边大型楼盘；
  2. 平台预售型商品，待货物备货完成时，再行按订单生成顺序依次安排配送，集中在基地配送日进行配送；
  3. 当平台在配送区域范围建立自提点，顾客可以在提交订单时，选择“上门自提”，选择离自己最近的提货地点，订单到达自提点后，系统发送短信告知顾客。
  4. 订单执行及配送情况，系统及时通过站内消息及手机短信形式告知客户，并在系统交易相关页面予以更新。

3.6.2 配送政策说明

  1. 由于业务发展变化、行政区域更名等因素，配送区域，可能会不时有扩大、变更或调整，也会增加配送的频次和缩短配货的间隔时间，如配送日增加周一、周四，也可能一周增加到三次配送（周一、三、五 或 周二、四、六），具体内容请以“一块地”平台最新公示为准。
  2. 生鲜商品属于鲜活易腐类特殊商品，不支持拒收，也不存在退货和换货业务，如顾客收货时发现问题，为尽快处理售后问题，需联系客服，提交相应图文凭证，走售后服务流程进行处理，一般是补发或退款。
  3. 顾客应当在订单中向商家提供准确的收货地址和收货人信息。顾客在提供收货人信息时，可以选择本人或者他人作为收货人。顾客选择他人作为收货人，该收货人违反买卖双方约定义务的，由顾客承担相应责任。
  4. 顾客需要变更订单中的收货地址或收货人信息的，应当在商品发出前与卖家取得联系征得卖家的明确同意，或协商进行处理。因顾客填写的收货地址和(或)收货人信息不准确，或者未经卖家同意要求变更收货地址或收货人信息，导致卖家发货后商品无法送达的，运费由消费者承担。
  5. 由顾客的疏忽导致的宅配货物不能签收产生退货和生鲜货物由于在途时间过长导到处出现腐烂等质量问题，应由用户承担其损失。

3.6.3 功能规划

针对开通“种植服务合约计划”的用户，其合约中有两项服务包产品涉及配送计划 1. 专属菜地代耕代种定制蔬菜育成产出 2. 日常配套家常大众蔬菜食材包

专属菜地的产出根据不同的作物品种其产出量差额比较大，每次采收前的预估数量偏差较大，另经过分拣和包装环节，损耗也比较大。

日常配套家常蔬菜食材包其一年的配额为360斤，分成100份进行配送，即每次配送一份（大于3.6斤，5～6个品种），顾客可以按需要增加配送量，按份累加，最多4份，

其中“专属菜地代耕代种定制蔬菜育成产出”只有在蔬菜成熟采摘期才会涉及配送服务，每周均有配送的是“日常配套家常大众蔬菜食材包”。

关于会员宅配，针对每种服务计划（套餐），会有几个全局配送服务的状态：

  • 未开启——这是初始状态；
  • 正常配送——表示客户已进行配送状态；
  • 暂停配送——表示客户已提出服务暂停需求，所有已生成的配送单状态应设置成“停配”，还未生成的后续配送计划单不再生成，直到客户进行恢复配送操作；
  • 挂起——一般是由于灾害及生产问题，无法正常提供服务，挂起，待情况改善后，再行提供正常
  • 冻结——由后台生管和配送管理人员进行设定，一般原因，是客户未续费并超期，将其剩余的配额挂起，直到客户续费，再接着配送
  • 结束——表示该配送服务计划完结，并不考虑客户是否还有配额。

3.6.3.1 顾客用例

开启配送

  • 当顾客参与某一种“种植合约”或“宅配合约”后，服务期开始计时，由于用户个人信息未完善，特别是地址信息未填写完全或不正确，因此需要顾客主动发起开始配送。因此可以理解这是一信号，将由系统引导顾客完善相关信息。具体见“生成配送计划”用例。

  • 这是一个全局状态，针对当前合约的配送

暂停配送

  • 顾客可以暂停配送，意味着客户后续已生成的计划配送单失效（即置为“停配”状态），且未生成的计划配送单，系统也不再生成，直到客户手动恢复配送。

  • 这是一个全局状态，针对当前合约的配送

恢复配送

  • 顾客可以对已暂停的配送，恢复成正常配送状态。

  • 这是一个全局状态，针对当前合约的配送

冻结配送

  • 顾客的配送出现异常，或者是公司政策调整，针对顾客服务到期后还有大量的配送包余额，运营人员可以将其挂起，待其续费后再正常进行配送。在此状态下，计划单不在生成，已有的计划单作废。

  • 这是一个全局状态，针对当前合约的配送

结束配送

  • 当顾客中止服务合约或配送内容提前使用完，配送需要关闭，由运营人员结束该会员的配送。

自提

  • 当顾客前往基地考察游玩时，可以将自有专属的产出和日常配套包提取，需要走出库流程，系统应记录，并扣减该会员的余额，特别是顾客自提专属菜地产出的东西是会影响最近一次的专属产出配送的。出库的时候应该修改其配送计划的数量。
  • 另一种方式是顾客可以在对近期的配送计划单调整为停配（或自提完成）。

自提点自提

  • 自提是平台为方便顾客收货，为顾客提供的一种固定地点提货的模式，顾客在提交订单时，选择“上门自提”，选择离自己最近的提货地点，订单到达自提点后，系统会以短信形式告知顾客；
  • 目前部分幼儿园可做为自提点，在没有实现自提点签收功能前，到自提点的货物，默认为配送完成。
  • 未来将拓展自提点并实现自提点的管理，包括货物的出入管理和退换货。
  • 自提点自提，每单的配送费可以视政策进行调整，鼓励顾客有用自提的方式

生成配送计划

  • 当顾客参与“种植服务合约计划”，初次开通服务时，开启配送时，须确定配送频次、配送日，配送时段、收货人等信息后生成整个1个月的的配送计划单，商家根据配送计划安排蔬菜的生产采收分拣包装和送货上门服务。
  • 顾客可以对已生成的配送计划进行查看和管理，配送计划应提供直观的界面（日历）形式予以呈现。建议采用日历的方式应能方便在周历和月历之间切换，并提供便捷的方式查看上周（月）或下周（月）的配送计划。
  • 当本次配送完成后，即物流跟踪数据反馈（人工或第三方数据接口）更新后，系统应将顾客宅配包余额减去本次配送的数量并更新，实时反应在相关页面上，与此同时，根据是否还有剩余的包，系统依据客户上次设置配送计划的条件生成一条新的配送计划单，月历上应实时刷新以反应这种变化。

重新生成配送计划

  • 当顾客对当前的计划不满意或有较大的调整时（如：搬家导致地址变换、有多个配送地址需频繁切换、合约升级从低配升级到高配），可以重新生成配送计划，原有的未执行的计划将全部清除。其它内容与“生成配送计划”相同。

更改收货地址

  • 顾客可以修改已生成的计划配送单上的地址，选择其它的地址，如果没有，则新建一条。每个地址应有别名，如：家、公司，地址A，地址B，方便在月历上显示，建议H表示Home（家），C表示Company(公司)，家庭会员地址簿应不超过26个（企业用户地址簿不受此数限制）。
  • 收货地址信息包括：收件人姓名、电话号码、行政区域及门牌号码。
  • 注意：要检测顾客配送地址是否超出配送区域，如有此情况，则要求客户更改，拒不更改的，则将此计划单，设为“无效”状态。

停止单个配送计划

  • 顾客可以将单个计划配送单暂停。前提是该计划单未进入执行状态，默认配送日前一天17:00后不允许顾客修改次日的配送计划中。将其状态设为“已停配”。如果该计划已进入执行做准备状态或执行状态，系统（或人工）应在这个可配置的时间点上将次日未停配的计划单设为“执行准备中”。
  • 专属菜地定制产出的蔬菜应兑换成菜金到顾客账户上。
  • 计划配送单的状态有:
  • 正常（新建、已添加数量、已变更地址）、
  • 停配
  • 停配已确认，这是一种结束状态
  • 配送计划已确认
  • 计划执行中，这种状态有多个子状态，如：已安排采、已开始分拣、已开始包装、已出库、第三方配送已出发、到达XX仓、到达XXX仓……
  • 取消配送，配送计划确认后，但在出库前，由用户提出，客服人员处理，取消此次配送，做出库异常处理，该货物照常扣减客户资产余额，但货物实际不发出。
  • 中止执行，这是一种结束状态，如：基地遭受自然灾害，无法覆行计划配送。应批量处理受影响的配送单（包括后续配送单的生成）
  • 作废，本状态相当于这个计划配送单是删除状态
  • 完成，这是一种结束状态

停止多个配送计划

  • 同“停止单个配送计划”一样，区别仅在于可以停止多个，一般场景是停止指定日（含）以后多次或全部的配送计划，将这连续的多次配送计划设为“停配”状态，如果要停配多个不连续的计划配送，应多次使用“停止单个配送计划”场景。

恢复单个已停止的配送计划

  • 顾客可以在17:00前对次日（当次）已停配的计划单恢复到正常状态
  • 顾客可以对未到期配送的已做停配处理的计划单恢复到配送装态。

恢复多个已停止的配送计划

  • 同“恢复单个已停止的配送计划”一样，区别在于是对多个已停止的配送计划单恢复到正常状态。

增减配送内容

  • 顾客可以对配送的蔬菜包数量进行增减，日常宅配默认每次一个包，顾客可以增加到多个，提前将配额消耗。（对应场景，周末，顾客开party，一个包不够招待，需要将未来的日常配送包提前使用）。这个数量不能为0，如果为0，相当于停配。这个数量也不过大，导致基地供应不上。这个值是可以在后台进行配置。建议取值为4。

忌口及偏好

  • 部分顾客对于蔬菜的品种有个人偏好，比如不吃辣椒，用户可以将某类蔬菜加入“忌口”列表，那么这些品种的蔬菜将不再配送到该客户手中。
  • 由于客户可能有多个配送地址，有些地址可能是家人和朋友，故可以添加例外，忽略“忌口”这个选项。

3.6.3.2 运营管理人员用例

顾客大部分场景及用例，后台运营人员均可代其操作，下面列出的用例是运营、客服、库管人员所有：

延期配送

  • 因自然灾害或恶劣天气、或交通管制，导致配送服务延期履行，应将所有当期计划进行变更或调整，并通过短信和站内消息的方式告知客户。

挂起

  • 因自然灾害或生产原因，导致服务无法履行，应将所有当期计划进行变更或调整，并通过短信和站内消息的方式告知客户。
  • 这种状态可能影响单一批次的配送，也可能是影响较长一段时间的。如果是长期影响，会影响运营，相关政策及办法需细化。？？？？？？？？？

生成工单

  • 在配送日前一天，要锁定所有的配送计划单，所有顾客当期“停配”和“正常”状态的计划配送单即将进入履行阶段，顾客不能再更改。系统或人工将“停配”的计划单转为“停配已确认”状态、“正常”的计划单转为“配送计划已确认”状态。

运费收取标准

订单中的生鲜商品和非生鲜商品，分别计算金额、运费，互不参与运费凑单；订单总计应收运费等于订单中的生鲜商品应付运费与非生鲜商品应付运费之和。

运费的计费是同一计价区内，根据出发站点与目的地，不同的运输方式，根据重量（体积根据公式折算成重量），首重价格，续重进行收费。

体积重量是一种反映包裹密度的计算方式。低密度的包裹，与实际重量比较，占用的空间通常较大。计算出来的体积重量与实际重量比较，取大者作为计费重量，用以计算运费。

折算成重量的办法做为计算运费的重量，称为体积重量或材积。

顺风快递国内业务参考 1. 同城、省内及经济区域内互寄：体积重量(KG)的计算方法为：长度(CM)x宽度(CM)x高度(CM)÷12000。 2. 省外非经济区域内互寄，体积重量(KG)的计算方法为：【即日/次晨/次日】长度(CM)x宽度(CM)x高度(CM)÷6000；【隔日】长度(CM)x宽度(CM)x高度(CM)÷12000。

备注：经济区域为：京津冀区域、江浙沪皖区域、黑吉辽区域、川渝区域。

“一块地”的会员在服务有效内，其服务合约内包含有满足日常食材宅配，额外购买的其它商品，则需要另外支付运费。 当前一块地的第三方物流合作方是“中诺”，其同城收费标准为： 每票（即每张单子）首重4公斤10元（含商品销售包装的重量），超过4公斤的，超出重量按1元/kg另外加收续重运费；中诺允许一次同一地址的几箱货算做一票。 当有2件货要配送给一个人时，包装上建议单号的末位采用2X1、2X2的形式， 如果有3件货，单号末位为3X1、3X2、3X3，以方便送货人配送准确。
